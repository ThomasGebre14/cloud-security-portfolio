# 🔒 Encryption, Hashing, and GRC

## Encryption
Encryption protects data by making it unreadable to unauthorized users.

### Symmetric encryption
- same key encrypts and decrypts

### Asymmetric encryption
- uses a public/private key pair
- common in HTTPS and digital signing

## Data Protection States
### Data at rest
Stored data on disks, databases, and storage accounts.

### Data in transit
Data moving over networks. HTTPS is a common example.

### Data in use
Data being processed in memory or CPU, sometimes protected through enclaves.

## Hashing
Hashing converts input into a fixed-length value.
- same input = same output
- not reversible like encryption
- often used for passwords

### Salting
Adding random data before hashing helps defend against dictionary and rainbow-table attacks.

## Governance, Risk, and Compliance (GRC)
### Governance
Rules, processes, and controls that direct how the organization operates.

### Risk
Identifying, assessing, and responding to internal and external threats.

### Compliance
Meeting legal, regulatory, and industry requirements.

## Related Compliance Concepts
- Data residency
- Data sovereignty
- Data privacy
